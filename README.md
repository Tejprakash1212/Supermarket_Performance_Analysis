Supermarket Sales Performance Analysis
🔍 Overview
This project presents an end-to-end data analytics workflow, from raw data processing to an interactive Power BI dashboard. It focuses on uncovering sales trends, customer behavior, and branch performance using Python, SQL, and Power BI.

📁 Dataset
Records: 1,000 transactions
Features: 17 columns (numerical, categorical, date/time)
Key Fields: Unit Price, Quantity, Total, Rating, Branch, City, Payment
Data Quality: No missing values or duplicates


🛠️ Tools & Technologies
Python: pandas, numpy, matplotlib, seaborn (EDA & cleaning)
SQL: PostgreSQL / MySQL / SQL Server (window & aggregate functions)
Power BI: Dashboard & visualization
DAX: KPIs and calculated measures

⚙️ Project Steps
1. Data Loading & Cleaning
Checked structure using info() and describe()
Handled missing values & duplicates
Standardized column names

2. Exploratory Data Analysis (EDA)
Analyzed sales, ratings, and product distributions
Identified trends and outliers

3. SQL Analysis
Used window & aggregate functions
Key insights:
Total sales & quantity
Revenue by branch
Customer demographics
Payment preferences

4. Power BI Dashboard
Built interactive dashboard
Created KPIs using DAX
Visualized revenue, trends, and performance

📊 Dashboard
Revenue & sales trends
Branch performance comparison
Customer segmentation
Product line analysis

📈 Key Insights
Sales are right-skewed with few high-value transactions
Branch C has the highest revenue and ratings
Customer distribution is nearly equal across genders
Digital payments are widely preferred
Product lines show consistent performance
